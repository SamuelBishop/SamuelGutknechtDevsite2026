# De-AI Copy Skill

When invoked, audit all user-facing copy in the project.

Rewrite copy to feel:

- specific
- human
- concise
- grounded
- plainspoken
- slightly personal
- confident but not salesy

Avoid:

- "leverage"
- "seamless"
- "robust"
- "cutting-edge"
- "passionate about"
- "crafting experiences"
- "journey"
- "empowering"
- "transformative"
- "innovative solutions"
- excessive em dashes
- generic senior-engineer boilerplate
- fake humility
- generic AI portfolio phrasing

Preserve:

- factual accuracy
- section intent
- user’s personality
- technical credibility
- understated senior-engineer tone
- outdoors/trail-running references only where natural

Process:

1. Find all user-facing copy in pages, components, markdown, and config.
2. Identify AI-sounding copy before rewriting.
3. Rewrite only the copy unless explicitly asked to adjust layout.
4. Keep headings short and concrete.
5. Prefer real details over adjectives.
6. After editing, summarize the biggest tone changes.
